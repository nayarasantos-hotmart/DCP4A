Ator
Usuário
Envia mensagem na IDE: "Configure essa conexão para mim: github.com/Hotmart-Org/mcp-hotmart-data"

Monta e salva a configuração do MCP Server no arquivo correto da IDE
Usuário

Frontstage

Tela de autenticação Okta no browser (padrão corporativo) com redirecionamento automático de volta para a IDE após login
"Pronto! Você está conectado aos dados da Hotmart. Datasources disponíveis: DHI_DATA_GOLD, DHI_DATA_DIAMOND. Quer explorar algum dado?"
"O datasource X contém dados de retenção por cohort. Quer que eu solicite acesso para você?"

Backstage
IA acessa o repositório no GitHub, lê o README, identifica que é um MCP Server e extrai a configuração necessária (URL do servidor, protocolo, variáveis de ambiente)
IA escreve o bloco de configuração no mcp.json (Kiro) ou claude_desktop_config.json (Claude Desktop), apontando para o MCP Server remoto de dados
MCP Server remoto recebe o token OAuth2 do Okta, valida identidade, emite access token + refresh token, armazena credenciais de forma segura (keychain do sistema)
MCP Auto-Config API identifica o role/time do usuário (via claims do Okta), consulta o Profile-to-Datasource Mapping, retorna lista de datasources autorizados
IA chama a Access Request API que cria ticket automático no Jira, notifica o owner do datasource, e registra a solicitação com status "pendente"
Access Request API recebe callback de aprovação do Jira, atualiza a lista de datasources disponíveis para o usuário no MCP Server
Jira workflow de aprovação aciona callback
